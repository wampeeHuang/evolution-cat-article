# 终审：机械质检 + 编辑判断

> v4.0 — 拆为两层：5a 机械质检（Ben Smith 5 项）+ 5b 编辑判断（Ben Smith + PG 双签）。
> 5a 是工厂质检。5b 是主编判断。两个都说行才是真行。

---

## 5a：机械质检（spawn Ben Smith）

> 工厂质检——尺寸对吗、标签贴了吗。不判断文章值不值得发。

### 检查清单

| # | 检查项 | PASS 标准 | 不通过 |
|---|--------|----------|--------|
| **1** | **反向映射** | 文中每个具体事实（数字/人名/时间/案例名称/实验描述/引语）可反向映射到素材包具体来源。未映射 = AI 编造 | 标疑或删除 |
| 2 | AI 味 | humanizer-zh 跑过，高频词 0 命中 | 回 Step 4 |
| 3 | 字数 | 2000-6000 | 回 Step 3 |
| 4 | 参考来源 | 逐条 URL 可访问 | 删除死链 |
| 5 | 落盘 | md + 封面 + 配图文件确认 | 补文件 |

### 第 1 项：反向映射操作

做一张对照表：

| 文章中的事实声明 | 素材包来源 | 状态 |
|-----------------|-----------|------|
| "126000 次对话" | PNAS / blockchain.news 报道 | ✅ |
| "35.3% → 51.3%" | 同上 | ✅ |
| "研究者先让模型合成香兰素……" | Ars Technica / SSRN 预印本 | ✅ |
| 某个找不到来源的叙述... | — | ❌ 删除 |

右列为空的 → AI 自己补的 → 删除。

### 可保证与不可保证

**拦得住**：AI 在素材包之外自己编造的事实、案例、细节、引语。

**拦不住**：素材包本身的错误。源数据错了文章跟着错——反mapping只能保"不超出素材"，不能保"素材没错"。

### 迭代纪律

- 每项不通过 → 回对应步骤修复，最多 **2 轮**
- 2 轮后仍有未达标 → 分级：
  - 🔴 第 1 项反向映射失败 / 死链 → 必须修复
  - 🟡 字数微偏 → 标注交付（Step 5b 可 override）
  - 🟢 配图仅 txt → 标记"待补"，不阻塞

**输出文件**：`_runtime/mechanical-qc-report.md`（5 项逐条 PASS/FAIL）

## 5b：编辑判断（spawn Ben Smith + PG，双签）

> 主编判断——值不值得发。这是杀稿门禁。不是机械清单。

### 前置

`_runtime/mechanical-qc-report.md` 存在且 5 项全 PASS → 进入 5b。不存则硬阻断。

### 并行 spawn

#### Ben Smith — 外部价值判断

输入：draft-v3.md + creative-brief.md + surprise-findings.md

| # | 检查项 | PASS 标准 | 为什么重要 |
|---|--------|----------|-----------|
| **①** | **Signal/Noise** | 「这篇东西，读者在别处能拿到吗？」不能 → PASS。能 → **FAIL** | 这是杀稿第一理由。如果你的文章只是聚合了别人也能搜到的信息，写得再好也是噪音 |
| **②** | **承诺兑现** | Beast 标题炸的点，正文兑现了吗？壳（标题）和核（surprise-findings）对齐了吗？ | 标题诈骗 = 读者再也不会回来。这是 Ben Smith 说的 Identity Hook, Substance Body |
| **③** | **反面论证** | 文章有没有展示最有力的反对论证？不找稻草人——找你真正尊重但不同意的观点 | 装全能丢信任。Semaform 的核心：分离事实和分析，展示反对观点 |
| ④ | 上帝之声 | 有没有"专家分析""深度解读""毋庸置疑""众所周知"等装全知全能的句式？ | 「记者不是算法，不是上帝之声」 |
| ⑤ | 身份钩子 | 读者转发时在对朋友说什么？正文配得上这个身份承诺吗？ | 被转发 = 读者在对自己社交圈说"看，我是这样的人" |
| **⑥** | **读者读感** | 文章的口吻、节奏、用典、情绪起伏是否匹配读者画像？裁判自问：读者读完全文，会觉得"这篇是写给我的"还是"这篇翻译自英文tech blog"？ | 口吻错位是最隐蔽的质量杀手——事实对、逻辑通、但读者觉得不是写给他的 → 划走。检查：文中有没有把读者熟悉的东西当陌生事物介绍？有没有默认读者共享非中文互联网的文化背景？ |

**输出文件**：`_runtime/editorial-judgment-bs.md`（逐项 PASS/FAIL + 杀稿建议）

#### Paul Graham — 内部逻辑判断

输入：draft-v3.md + outline-review.md + surprise-findings.md

| # | 检查项 | PASS 标准 | 为什么重要 |
|---|--------|----------|-----------|
| **①** | **递进链还在吗** | 写的过程中有没有偏离 PG 大纲？section 之间递进关系还在不在？ | Sugarman 的滑滑梯可能滑偏——写着写着就偏离了 original outline 的逻辑链 |
| **②** | **读者能说出 3 个收获吗** | 读完觉得"学到了三个东西"还是"看完了"？ | 如果读者只能说"写得不错"但说不出学到了什么——这篇是滑梯没有内容 |
| **③** | **Surprising Idea 被写淡了吗** | surprise-findings 里的核心意外，在终稿里的冲击力还在不在？ | 最常见的衰变：大纲时意外很 sharp → Sugarman 写完全文 → 意外被滑滑梯稀释了 |
| ④ | 逻辑断层 | 终稿实际文本层面，N 节到 N+1 节读者会不会断？ | 大纲层面的断层 PG 在 Step 2.5 已查——这里查实际文本 |

**输出文件**：`_runtime/editorial-judgment-pg.md`（逐项 PASS/FAIL + 修改建议）

### 双签判决

| Ben Smith 结果 | PG 结果 | 动作 |
|---------------|---------|------|
| PASS | PASS | ✅ 发 |
| **FAIL（①②③任一）** | — | **🔴 杀稿。** Signal/Noise FAIL = 噪音。承诺兑现 FAIL = 标题诈骗。反面论证 FAIL = 装上帝。这些修不了——是选题和角度层面的问题。 |
| PASS（①④⑤全PASS）| FAIL | 回 Step 3 Sugarman 重写（PG 的诊断给 Sugarman），最多 **1 轮** |
| ④或⑤ FAIL（①②③全PASS）| PASS | 回对应步骤修（④→改语气/⑤→改开头），最多 1 轮 |

### 杀稿 ≠ 失败

杀稿是管线设计功能，不是 bug。不发一篇烂文章 > 发十篇平庸文章。

杀稿后动作：
- 素材包保留（下次相关选题可复用）
- surprise-findings 写入选题库备注（"此选题搜完无意外——2026-XX-XX"）——避免下次重复踩坑
- 如果 surprise-findings 里有部分亮点但整体撑不起一篇 → 标记"碎片待拼"——等有相关素材时再拼
