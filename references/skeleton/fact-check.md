# 事实校验

> Step 1 执行。骨架层，不可通过 retro 回流修改。
> 职责：源验证——素材包的数据能不能用。

---

## 一、事实校验铁律

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | 禁止使用模型生成的虚假链接 | 发布后 404 → 信任崩塌 |
| 2 | 禁止保留无法访问的死链 | 同上 |
| 3 | 禁止标注"待确认"后照常发布 | "待确认"= 没确认 = 不准写 |
| 4 | 无法验证的内容必须删除 | 不留、不标、不凑数 |
| 5 | 多来源论文的数据不可混用为同一研究的发现 | 预印本和正式发表各自标注 |

## 二、源质量门禁

单条判断，不是复杂流程：

1. 核心事实 ≥3 个独立来源印证
2. 核心论据依赖的关键数据点有交叉验证（见 source-trace.md §二 交叉验证卡）。单源数据可保留但必须标注"该数据仅来自 XX，未经第三方确认"，且不能作为文章唯一支柱论据。

→ 不通过则扩大搜索或放弃选题。

## 三、数字核验路径

| 数据类型 | 核验路径 |
|---------|---------|
| Stars/Forks/Issues | GitHub API |
| 价格/版本 | 官网 |
| Agent/Plugin 数量 | 官方文档或 API |
| 用户量/下载量 | 官方发布 + 第三方平台交叉验证 |

核验结果标注：
- ✅ 可追溯 — 有公开来源可查证
- ✅ 一手 — 亲自验证
- 🔮 推测 — 基于已知信息的推断（必须标注，不可当事实发布）
