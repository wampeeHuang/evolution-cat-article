## 一、事实校验铁律

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | 禁止使用模型生成的虚假链接 | 发布后 404 → 信任崩塌 |
| 2 | 禁止保留无法访问的死链 | 同上 |
| 3 | 禁止标注"待确认"后照常发布 | "待确认"= 没确认 = 不准写 |
| 4 | 无法验证的内容必须删除 | 不留、不标、不凑数 |
| 5 | 多来源论文的数据不可混用为同一研究的发现 | 预印本和正式发表各自标注 |

## 二、源质量门禁

单条判断，不是复杂流程：

1. 核心事实 ≥3 个独立来源印证
2. 核心论据依赖的关键数据点有交叉验证（见 source-trace.md §二 交叉验证卡）。单源数据可保留但必须标注"该数据仅来自 XX，未经第三方确认"，且不能作为文章唯一支柱论据。

→ 不通过则扩大搜索或放弃选题。

## 三、对比公平性校验

> v7.0.1 新增。盲区发现：核查"数字有没有来源"不够——还要查"对比对象有没有代表性"。

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | **同档对齐**：对比必须在同类/同定位产品之间。旗舰比旗舰，轻量比轻量。拿 Flash 比 Opus 不行 | 数字有来源但结论失实 |
| 2 | **时效性**：参与对比的产品必须是当前可用/在售版本。已下架（如 GPT-4o）、已禁用、已归档版本不得作为核心论据 | 对比是博物馆陈列，不是市场现状 |
| 3 | **地域可用性**：区域禁用的产品（如 Claude Fable 5 仅美国可用）不得作为面向中国读者的价格参考锚点 | 读者用不了，数字无意义 |
| 4 | **极值不单独引**：如果同一组对比中有一个极端值（如最高价出现在已禁用产品），不得单独引用它作为"XX倍"的唯一锚点 | cherry-picking 以偏概全 |
| 5 | **口径一致**：价格对比必须用同一计费单位（每百万token输出价），不同模型的价格来源标注采集日期 | 时间/口径不一致，比出来的倍数失真 |

核查结果标注：
- ⚖️ 公平对比 — 同档对齐 + 当前可用 + 地域可访问
- ⚠️ 有限代表 — 有 1 项不满足（如仅可比产品有限），需在正文标注"对比限于XX范围"
- ❌ 失实 — ≥2 项不满足 → 必须修正或删除对比

## 四、数字核验路径

| 数据类型 | 核验路径 |
|---------|---------|
| Stars/Forks/Issues | GitHub API |
| 价格/版本 | 官网 |

| 用户量/下载量 | 官方发布 + 第三方平台交叉验证 |

核验结果标注：
- ✅ 可追溯 — 有公开来源可查证
- ✅ 一手 — 亲自验证
- 🔮 推测 — 基于已知信息的推断（必须标注，不可当事实发布）
