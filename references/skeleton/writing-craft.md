# 写作技法

> 骨架层 — 通用写作工程技法，不可通过 retro 修改。个人风格偏好看 flesh/ 层。

---

## 一、AI 角色边界

写作者是人，AI 是文风生成器，不是替代思考的工具。

### AI 适合做的

| 能力 | 说明 |
|------|------|
| **找证据和佐证** | 给出观点，AI 在历史/学术/文化中找支撑（和反驳）的论据 |
| **找类比和比喻** | 提供多个候选类比解释抽象概念，人做最终选择 |
| **按确定角度扩写** | 人定好核心角度和段落标题，AI 填充论述和细节 |
| **补充学科背景** | 格式塔心理学、因果语言模型原理等准确表述 |
| **梳理逻辑结构** | 调整段落顺序、检查逻辑流畅度 |

### 人必须亲自做的

| 能力 | 为什么 AI 做会暴露 |
|------|-------------------|
| **第一手观察和真实经历** | 无法编造，一编就假 |
| **核心创意角度** | AI 可提供候选，但"对，就是这个！"的判断只有人能下 |
| **情绪的真实表达** | AI 倾向知识性描述（"我很震撼"），人写体感记忆（"我当时就愣住了"） |
| **数据→人物的同理心转换** | 从一个数字想象出一个完整人生的温度，需要作者真心感受 |

### 理想协作流

```
人：素材 + 核心观点 + 个人经历 + 情绪节点
 ↓
AI：补充背景知识 + 找证据类比 + 结构建议 + 按角度扩写
 ↓
人：二次改写（加入自己的声音、打破节奏、补充真实细节）
 ↓
AI：四层质检 → 输出修改建议
 ↓
人：终审定稿
```

## 二、22 个通用写作技法

### 1. 节奏感

句子时长时短，用逗号制造口语化停顿。段落间跳跃自然，一句话独立成段制造重点。

节奏的本质是推进系统。好的节奏像波动——每次围绕主线偏出去一点，让读者喘口气、长点见识、看个案例，再用一句话拉回来。偏出去的时候习惯性加入"扣主线句"，一句就够但必须高频出现。

### 2. 论述中的故意打破

在展开观点时，故意加入口语打断破坏严谨性：重复强调、中途叹息、省略主语、刻意模糊。这些不是错误，是让文章听起来像真人在说话。

### 3. 知识输出方式

知识是"聊着聊着顺手掏出来"的，不是"下面我来科普一下"。让读者觉得你脑子里本来就有这些东西，正好跟眼前的事对上了。

### 4. 私人视角

用"我也面临这个问题"连接个人经历和公共议题，不是"这给我们的启示是"。从自己的真实经历切入。

### 5. 判断力

敢下判断，有明确好恶。但表达姿态是"我被打动了""我觉得他说的是事实"——承认自己被影响的姿态，不是居高临下的点评。

### 6. 对立面的理解与承认

讲观点前，先站在对方角度把处境具体化，承认其合理性（"我非常理解这种感觉"），再切入自己的不同视角。读者觉得"他理解我"，观点才有说服力。

### 7. 情绪表达

用体感记忆代替知识性描述。"我当时就愣住了" > "我感到震撼"。"鼻子一酸" > "我很感动"。情绪标点（"...""??? ""= ="）是情绪的具象化，可按个人风格使用。

### 8. 亲自下场

最核心的写作基因。不是隔空评论，是真的去做那件事。读者感觉到"这个人真的做了"，不是"这个人在想象做这件事"。

### 9. 人物画像法

从一个数据点出发，用极短篇幅想象背后那个具体的人的完整人生：

```
触发点数据（"1000个已付款"）
→ 快速代入（"他可能是一个……"）
→ 多维堆砌（城市→职位→生活→心理→具体行为）
→ 情感锚定（为什么这样做）
→ 细节具象化（十几平米出租屋、拼好饭）
```

3-5 句话内让这个人物立体，产生共鸣。

### 10. 文化升维

聊完具体事情后，连接到一个更大的文化/哲学/历史参照物。不是硬凑的升华，是"聊着聊着自然想到了"的感觉。

### 11. 句式断裂

极短句子或短语独立成段，制造停顿和重量感。不能滥用，关键节点用一下效果极强。

### 12. 回环呼应（契诃夫之枪）

第一幕挂了一把枪，第三幕它就得开火。前面埋的细节后面得响。文章内部要有 callback 结构——前面提到的意象或句子，在后面以变体形式再次出现。开头或中间留钩子，结尾 callback 回来，让文章从"信息流"变成"作品"。

### 13. 谦逊铺垫法

给观点或建议前，先用自谦降低读者防御。"我也不知道行不行""不成熟的经验"——这不是虚伪的谦虚，是真实的不确定感。尤其在方法论、教程类长文，开头和结尾都需要这种铺垫。

### 14. 读者直呼法

关键节点直接跟读者对话："你想想看""你也可以回想一下"。不是通篇用，在需要拉近距离或要求读者行动时精准投放。

### 15. 疑问句的节奏作用

疑问句是节奏的刹车和转向。"为什么会这样？"——让读者停一秒，准备接收新信息。"听着很难理解对吧"——共鸣，"我用大白话举个例子"——承诺接下来会简化。

### 16. 层层剥开的修辞

不直接讲结论，用"现象 → 表面解释 → 更深的追问 → 核心洞察"展开。让读者参与思考过程，感受推理路径，不是被动接收结论。

### 17. 英雄之旅叙事弧

结构：遇到具体问题/好奇 → 一步步去做、踩坑 → 秀出"卧槽"的结果。适用于调查实验型和产品体验型文章。冒险起点必须是具体的、读者能代入的困境或好奇，不是抽象命题。

### 18. 反向论证

先满足读者期待，再打破它。力度控制："我也曾经这样想"的姿态，不是"你们都错了"。

### 19. 案例人物的公正性

用真实人物做案例时，讲完整故事弧，不只截取对论点有利的片段。片面截取让懂行的读者觉得你在偷换概念。

### 20. 领域语言真实性

涉及特定领域（游戏、代码、设计等）时，必须用真正圈内人才用的语言和细节。不懂的领域不装懂。

### 21. 逐一展示法（升番逻辑）

涉及多个产品/案例对比时，不一次性罗列结论。逐一展示，每个带一句吐槽或点评，制造发现感和节奏感。排列遵循升番逻辑：最弱放前面，最炸留到最后，中间要有"以为到顶了结果还能往上翻"的惊喜。

### 22. 创意案例的力量

涉及产品评测或工具推荐时，至少有一个让人"卧槽"的创意案例。包装成微型故事：亮出挑战 → 展示脑洞 → 秀出过程 → 引爆结果。

## 三、文章结构模板

```
【开头】感性切入，从一个具体事件/场景开始，迅速建立情绪
  ↓
【背景铺垫】聊天式科普，让非专业读者也能看懂
  ↓
【核心内容】分几个板块展开，每板块：
  - 一个明确观点
  - 至少一个具体场景/人物/对话支撑
  - 私人视角连接（"我自己也是这样"）
  - 一句扣主线的话
  ↓
【升华】从具体事件拉到更大的文化/哲学/历史参照物
  ↓
【收尾】选最合适的方式：
  - 引用收尾 / 哲思余韵（短句留白）/ 行动呼吁 / 回环呼应开头
```

## 四、四层质检体系

写完后从最基础的硬性规则到最主观的活人感判断，层层递进。只有四层全部通过，文章才算达标。

### L1 硬性规则（自动扫描）

- **禁用词扫描**：按 flesh/avoid-list.md 的个人禁区 + density-gate.md 的骨架禁区合并扫描
- **禁用标点扫描**：按 flesh/hard-preferences.md 的个人标点规则扫描
- **结构套话扫描**："在当今……的时代""随着……的发展""让我们来看看……"
- **空泛名称扫描**：所有工具/产品使用具体名称，不出现"AI工具""某个模型""相关技术"

通过标准：以上四项零命中。

### L2 风格一致性（模式匹配）

- 开头是否从具体事件/场景切入（不是宏大叙事）
- 是否有长短句交替
- 是否有一句话独立成段的断裂效果
- 偏离主线后是否有扣主线句拉回
- 是否避免了不必要的小标题
- 知识输出是否以聊天式呈现

### L3 内容质量（深度审查）

- 每个核心观点是否有具体场景/人物/数据支撑
- 是否有至少一处文化升维
- 在讲核心观点前是否有对对立面的理解和承认
- 创意案例是否包装成微型故事

### L4 活人感（最终人格层）

核心问题：**读完这篇文章，感觉是一个真人在认真聊一件打动他的事，还是一个 AI 在输出信息？**

四个感知维度：
- **温度感**：情绪表达是体感记忆还是知识性描述
- **独特性**：有没有"只有这个作者才会写出来的角度"
- **姿态**：是不是"有见识的普通人在认真聊天"，有没有滑入"导师教学"或"品牌营销"
- **心流**：从头到尾读，有没有注意力断掉、需要回头理解逻辑的地方

L4 FAIL → 第三道门禁杀稿。
