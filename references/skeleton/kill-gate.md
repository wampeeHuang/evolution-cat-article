# 杀稿门禁

> 骨架层 — 不可通过 retro 修改。管线三道门禁，统一收归于此。

---

## 三道门禁总览

| 门禁 | 阶段 | 触发时机 | 判定标准 | 结果 |
|------|------|---------|---------|------|
| **第一道** | 判 | 选题后、搜素材前 | 四问 | 3+🔴→杀 / ④🔴→杀 |
| **第二道** | 搜 | 搜完素材后、构结构前 | 意外发现 | 0意外→杀 |
| **标题门禁** | 写→审 | 输出标题候选时（阶段四），终审时复核（阶段五） | 公式匹配+主题标签检测+三问诚实 | 无候选通过→退回 |
| **F0 事实门禁** | 审 | 阶段五 L1 之前 | 事实核查三级标注 | ❌→硬阻断，修正后重过 |
| **第三道** | 审 | F0 PASS 后 | L4活人感 | FAIL→杀 |

---

## 一、第一道门禁：选题四问

在 Step 判 执行，搜素材前跑。

### ① 认知税

读者要先理解几个陌生概念，才能拿到洞察？

```
0-1 个 → 🟢 过
2 个   → 🟡 可写，但每个概念必须在文中用一个类比消解
≥3 个  → 🔴 杀。你不是在写AI文章，是在开专业课
```

### ② 洞察值

读者拿到最终洞察时，反应是"操原来可以这样"还是"嗯我知道"？

```
"操原来可以这样" → 🟢 过
"有点意思"（直觉模糊感知但未被清晰表达过）→ 🟡 可写，靠类比和反转拉高
"嗯我知道" → 🔴 杀。写出来是复习不是发现
```

### ③ 分享货币

读者转给朋友时，能靠转发完成自我表达吗？

```
转发=标榜"我懂了很厉害的东西" → 🟢 过
转发=说"这个对你有用" → 🟡 可写，实用型文章靠这个
读者自己都讲不清楚文章说了什么 → 🔴 杀。会点赞不会转发
```

### ④ 平台合规

这个话题/角度是否存在微信红线风险？

| 风险项 | 判定 | 进化猫高频触发场景 |
|--------|------|-------------------|
| **标题诈骗** — 标题承诺正文无法兑现 | 🔴 | "AI替代XX岗位""XX工具让效率提升10倍" |
| **夸大AI能力** — 声称AI能做到但无可靠证据 | 🔴 | "AI已经能自主创业""AI完全替代某职业" |
| **虚假数据** — 编造或引用不存在的研究/数字 | 🔴 | 素材包无来源的任何数字和百分比 |
| **诱导分享** — "转发给你的朋友""不转不是XX" | 🔴 | 结尾转发钩子越界的变体 |
| **敏感领域** — 未证实医疗/金融建议、政治敏感 | 🔴 | "AI诊断比医生准""用AI炒股稳赚" |
| **侵权** — 大段翻译/搬运外网付费内容 | 🟡 | 外网全文翻译需改写+标注来源 |
| **负面对比** — 点名贬低具体产品/公司 | 🟡 | 可客观对比，不可贬损 |
| 无风险 | 🟢 | — |

### 判定

| 红灯数 | 动作 |
|--------|------|
| 0 红灯 | 过 |
| 1 红灯 | 过，写作时针对性补强 |
| 2 红灯 | 退回。重新找角度，或降级 |
| **3+ 红灯** | **杀** |
| **④ 任一 🔴** | **杀或重定角度**（此条独立，前三问全绿但平台违规照样杀） |
| ④ 有 🟡 | 可写，第三道门禁时逐条复核 |

## 二、第二道门禁：意外发现

在 Step 搜 执行，搜完素材后、进入构结构前跑。

### 判定标准

```
搜完素材后，至少找到 1 个意外发现 → 🟢 过
搜完素材后，0 意外发现 → 🔴 杀
```

### 什么是意外发现

"意外发现"的定义：**搜之前你不知道的事实/数据/案例/视角，且能支撑或颠覆文章的核心主张。**

不是"搜到了相关文章"——那是素材收集。意外发现是搜完后的那声"我靠原来是这样"。

例：
- 搜某个AI工具的写作质量 → 意外发现：它的输出质量与 prompt 语言强相关，英文 prompt 得分比中文高 40% → 这可以成为文章的核心反转
- 搜某个行业数据 → 意外发现：表面上增长的数据，拆开看全是头部通吃，长尾在萎缩 → 这个视角本身值一篇文章

### 对抗"假意外"

AI 搜素材容易把"我没听过"当成"意外发现"。校验标准：

| 真意外 | 假意外 |
|--------|--------|
| 数据来自一手来源（论文/财报/官方公告） | 数据来自二手汇总（别人的文章/公众号） |
| 可以在文中作为锚点引用 | 引用出去读者会反问"这数据哪来的" |
| 与文章主张有因果/反转关系 | 只是相关，放在文章里像插播广告 |

至少一个意外发现来自一手来源，否则退回重搜。

### 意外值为 0 时

```
>> 退回。三个选择：
>> 1. 换搜索方向重新搜（不同关键词、不同数据库、不同语言）
>> 2. 重新审视选题——如果搜不出意外，大概率是共识型选题
>> 3. 杀稿。写入 .abandoned："此选题搜完无意外——YYYY-MM-DD"
>> 意外发现写入选题库备注，下次相关选题可跳过此方向
```

## 三、F0 事实门禁：后写作事实复核

在 Step 审 执行，L1 质检之前必须先过。这是硬阻断门禁。

### 判定标准

```
F0 事实复核 ALL PASS（无 ❌）→ 🟢 进入 L1
F0 有 ❌（事实错误/无法验证）→ 🔴 硬阻断，修正后重新过 F0
```

### 三级标注

| 标注 | 定义 | 动作 |
|------|------|------|
| ✅ 一手验证 | WebFetch 直接命中官方页面，数字一致 | 保留 |
| ⚠️ 二手佐证 | 一手源不可访问，≥2 个独立二手来源交叉一致 | 保留，正文标注来源局限性 |
| ❌ 无法验证/事实错误 | 找不到来源 / 数字与来源不符 | 无法验证→删除断言；事实错误→修正后重过 |

### 执行要求

- 从 draft-v1.md 逐条提取事实断言（数字/日期/研究结论/专有名词/产品名称）
- 每条断言用 WebFetch 抓取一手来源页面核验，不用 WebSearch 摘要替代
- 不能抓取的（付费墙/404）→ 至少 2 个独立二手来源交叉验证
- 如文章涉及产品/价格对比，运行对比公平性校验（fact-check.md §三）
- 输出 `_runtime/fact-verification.md`

### F0 不通过 → 动作

```
>> 硬阻断。不得进入 L1-L4。
>> 事实错误 → 修正 draft-v1.md 中对应断言 → 重新过 F0
>> 无法验证 → 删除断言或替换为可验证的 → 重新过 F0
>> F0 不过不进入 L1，不给退回——必须修正到过为止
```

## 四、第三道门禁：编辑判断

在 Step 审 执行，F0 事实复核 PASS 后，L4 活人感质检完成后跑。

### 判定标准

```
F0 事实复核 PASS（前置条件，不满足则硬阻断）
L4 全部维度 PASS → 🟢 过
L4 任一维度 FAIL → 🔴 杀（或退回重写，最多一次）
```

L4 活人感 = 15 维风格指纹校验，定义在 `flesh/voice-signals.md`。机械规则（错别字、格式、事实）已在审阶段前两步扫过，第三道门禁只审一件事：**读起来像活人写的吗？**

### 两个一票否决项

| 否决项 | 定义 | 检测方式 |
|--------|------|---------|
| **Signal/Noise** | 删掉这句话，文章损失什么？段落中每句都通过此测试。发现"没错但也无所谓"的句子 ≥ 3 处 → FAIL | 逐句过 |
| **上帝之声** | "值得注意的是""毋庸置疑""众所周知""我们应该" → 文中出现 ≥ 2 处 → FAIL | 全文搜索 |

这两项是 L4 中的硬指标——任一 FAIL，整篇杀，不给退回重写机会。原因：Signal/Noise 和上帝之声不是局部问题，是行文基调问题，重写等于重写全文。

### 杀稿后动作

```
>> 在文章目录 touch .abandoned
>> 写入杀稿原因（门禁编号 + 红灯明细 + 简短说明）
>> 素材包保留（下次相关选题可复用）
>> 第四道门禁杀稿 → 无退回机会
>> 第五道门禁杀稿 → 可退回重写一次，再 FAIL 则杀
```

## 五、平台合规三道检查

| 检查点 | 对应门禁 | 方式 |
|--------|---------|------|
| **粗筛** | 第一道（判） | ④ 平台合规检查清单 → 🔴 则杀或重定角度 |
| **复核** | 第三道（审）机械层 | 文中的数字/断言是否有公开来源？AI能力声明是否夸大？ |
| **终审** | 第三道（审）编辑层 | 标题承诺是否兑现？引用内容是否侵权？有无诱导分享措辞？ |

## 六、杀稿示例

### 例一：控制工程三原则 × AI使用 — 第一道门禁 3🔴 → 杀

- 认知税 🔴：读者需理解"充分激励""逆响应""鲁棒性权衡"三个陌生术语
- 洞察值 🔴：最终洞察"用AI要建立反馈回路"——资深AI用户已在直觉性做
- 分享货币 🔴：读者转给朋友时讲不清"控制工程三原则是什么"
- 平台合规 🟢

**杀。** 源域和目标域之间认知距离超过读者接受半径。2026-06-04 实战：三版重写仍杀——选题可行性不是写作质量能弥补的。

### 例二：AI写作工具对比 — 第二道门禁 0意外 → 杀

搜了 8 篇公众号文章、3 篇少数派评测、2 个 Reddit 帖子。所有结论都是"Claude 长于创意、GPT 长于逻辑"的变体——共识型选题，搜不出意外发现。

**杀。** 标记：此方向为共识红海，除非有独家一手测试数据，否则不再碰。

### 例三：某AI工具深度评测 — 第三道门禁 Signal/Noise FAIL → 杀

文中出现：
- "在当今AI技术飞速发展的时代背景下"（0 signal）
- "值得一提的是，这款工具还支持……"（上帝之声）
- "总而言之，这是一款值得尝试的工具"（0 signal）

全文 5 处 0-signal 句 + 3 处上帝之声 → **杀。**
