# 读者契约

> 骨架层 — 不可通过 retro 修改

每篇文章跟读者签一份隐形合同：**花5分钟读这篇，你会得到______。** 合同不兑现 → 读者划走。合同模糊 → 读者不打开。

---

## 一、定义

读者契约 = 文章在读者脑子里挂的那个问号。从第一句开始挂，全文结束才放下。

问号不是标题党——问号是正文每一段都在回答的那个问题。读者读完最后一段时，必须能一口气说出自己得到了什么。

不是"这篇文章写了什么"——那是选题。是"读者读完后会觉得自己获得了什么"——那是契约。

---

## 二、八大契约类型

### 1. 类比传导 — "A和B底层一样，懂了B就不用学A的规则"

**挂的问号：** 这两个看起来完全无关的东西，怎么会是同一件事？

**心理引擎：** 熟悉+陌生并置 → "啊哈"时刻 → 跨域认知偷渡（"我把一个领域的智慧偷到了另一个"）

**结构：** 展示源域（熟悉的）→ 提炼通用原理 → 映射到目标域（你要讲的）→ 从类比反推反直觉洞察

**硬条件：** 源域和目标域有结构同构性，不是表面比喻。"AI就像一个人"= 不过。"恒温器的反馈回路和AI对话的反馈回路是同一种信息结构"= 过。

**进化猫适用性：** ★★★★★ 核心主力。非技术读者靠类比理解AI。

### 2. 认知反转 — "你以为X对，真正起作用的却是Y"

**挂的问号：** 我一直在做的事，是不是一直在伤害我？

**心理引擎：** 认知失调 → 阐释需求 → "比别人先觉醒"的分享欲

**结构：** 建立现有认知（"你是不是也这样"）→ 打破（意外数据/逻辑）→ 重建新认知 → 行动

**硬条件：** 反转必须经得起核查。不能反科学常识，只能反行业惯性认知。没有意外硬造 = 标题党。

**进化猫适用性：** ★★★★★ 最强记忆锚点。AI领域充满误解。

### 3. 问题方案 — "你有痛点，这里有解法"

**挂的问号：** 这个问题能解决吗？怎么解决？

**心理引擎：** 痛点命名（焦虑）→ 方案预期（奖励预期）→ 每步一个微奖励

**结构：** 场景共鸣 → 问题定性（"不是你的问题，是结构问题"）→ 3-5步方案 → 一个立刻能做的动作

**硬条件：** 方案必须具体、可立刻执行。"看完能用"是检验标准。

**进化猫适用性：** ★★★★★ 最直接的阅读动机。"我想用AI但用不好"。

### 4. 框架清单 — "你知道得太碎，这里有完整框架"

**挂的问号：** 我脑子里那些碎片，能被组织成一个什么东西？

**心理引擎：** 碎片焦虑 → 秩序渴望 → 计数完成 → 收藏安全感

**结构：** "关于X，你只需要知道这N个原则"→ 每原则：概念 + 为什么重要 + 例子

**硬条件：** N=3-7，条目互斥且穷尽（MECE）。"100个技巧"互相重叠 = 水货。

**进化猫适用性：** ★★★★ AI知识碎片化严重，框架稀缺。但单独使用容易"干"。

### 5. 故事见证 — "有人跟你一样困境，看他怎么走出来的"

**挂的问号：** 他能走出来吗？我能吗？

**心理引擎：** 共情 → 叙事传输（被故事吸入）→ 替代性学习

**结构：** 负面处境（具体场景）→ 转折（关键认识）→ 变化（做了什么不同）→ 结果 → 方法论提取

**硬条件：** 故事真实，细节具体。方法论提炼部分不能<故事部分的1/3。

**进化猫适用性：** ★★★★ 降低距离感。但进化猫是"方法论翻译"品牌，故事不能压倒方法论。

### 6. 悬疑解谜 — "有个反常现象，带你侦探式揭开"

**挂的问号：** 为什么会这样？答案到底是什么？

**心理引擎：** 信息缺口 → Zeigarnik效应 → 认知闭合需求

**结构（Cialdini 6步）：** 谜题 → 加深谜题 → 排除错误答案 → 线索 → 解谜 → 启示

**硬条件：** 现象本身够奇怪。普通到不用解释的现象不成立。

**进化猫适用性：** ★★★★ 适合"为什么AI会这样"类问题。但需要够奇怪的现象。

### 7. 共鸣确认 — "你的挫败有名字、有原因、能解决"

**挂的问号：** 只有我这样吗？这到底是什么？该怎么办？

**心理引擎：** 孤独焦虑 → 命名确认（"对！就是这种感觉！"）→ 归因释然 → 转发=自我表达

**结构：** 具象场景 → 情绪命名 → 原因拆解 → 出路（小而具体，不鸡汤）

**硬条件：** 场景极度具体（行为级），情绪广泛存在但未被清晰命名。纯共鸣无出路 = 贩卖焦虑。

**进化猫适用性：** ★★★ AI焦虑是真实情绪，但需找到新命名角度。可混合使用。

### 8. 趋势研判 — "你看到变化但没看到水面下的板块"

**挂的问号：** 到底在发生什么？往哪走？我该怎么准备？

**心理引擎：** 未来不确定性焦虑 → 认知地图渴望 → 决策安全感

**结构：** 表面现象 → 深层结构（可观测信号）→ 推演 → 行动框架（不是预测，是准备）

**硬条件：** 信号足。"XX必将取代YY"= 信用破产。"未来充满变化"= 说了等于没说。

**进化猫适用性：** ★★★ AI变化快适合趋势，但需要足够信号才能写。

## 三、选型决策

### 选型三问（搜素材前必答）

```
① 这个话题里最让我意外的是什么？
   → 有经得起核查的反转 → 认知反转
   → 有一个奇怪但真实的现象 → 悬疑解谜
   → 有两个领域底层一样 → 类比传导
   → 没有意外，但读者有痛 → 问题方案

② 读完这篇文章，读者会跟朋友怎么转述？
   → "原来X不是Y，而是Z" → 认知反转
   → "原来X和Y是一回事" → 类比传导
   → "原来这个问题的解法是这样" → 问题方案
   → "原来有人跟我一样/原来是这样走出来的" → 故事见证/共鸣确认
   → "原来这些碎片能这样组/原来趋势是这个方向" → 框架清单/趋势研判

③ 这篇文章的分享货币是什么？
   → "我知道了你不知道的" → 认知反转/悬疑解谜
   → "这个角度太妙了" → 类比传导
   → "这个对我有用" → 问题方案/框架清单
   → "我就是这样" → 共鸣确认
   → "跟我想的一样/帮我说清楚了" → 趋势研判
```

## 四、杀稿门禁

选题通过选型三问后，动手前再过一道杀稿门。三道全亮红灯 → 杀，不留情面。

### 杀稿三问

```
① 认知税 — 读者要先理解几个陌生概念，才能拿到洞察？
   → 0-1个陌生概念 → 🟢 过
   → 2个陌生概念 → 🟡 可写，但每个概念必须在文中用一个类比消解
   → ≥3个陌生概念 → 🔴 杀。你不是在写AI文章，是在开控制工程选修课

② 洞察值 — 读者拿到最终洞察时，反应是"操原来可以这样"还是"嗯我知道"？
   → "操原来可以这样" — 之前完全不知道 → 🟢 过
   → "有点意思" — 直觉模糊感知但没被清晰表达过 → 🟡 可写，靠类比和反转拉高
   → "嗯我知道" — 已经在这么做或已从别处看过 → 🔴 杀。写出来是复习不是发现

③ 分享货币 — 读者转给朋友时，能不能靠转发完成自我表达？
   → 转发=标榜"我懂了很厉害的东西" → 🟢 过
   → 转发=说"这个对你有用" → 🟡 可写，实用型文章靠这个
   → 读者自己都讲不清楚这篇文章说了什么 → 🔴 杀。他会点赞但不会转发
```

### 判定标准

| 红灯数 | 判定 |
|--------|------|
| 0 红灯 | 过，进入选型 |
| 1 红灯 | 过，但写作时必须针对性补强（认知税→加类比，洞察值→找反直觉角度，分享货币→简化核心主张到一句话） |
| 2 红灯 | 退回。重新找角度，或降级为轻输出 |
| **3 红灯** | **杀。不留。** |

### 杀稿示例

**控制工程三原则 × AI使用** — 3 红灯，杀。

- 认知税 🔴：读者需理解"充分激励""逆响应""鲁棒性权衡"三个陌生术语。即使每个都配了类比，读者要跳三次。对午休刷手机的人，跳一次是趣味，跳三次是作业。
- 洞察值 🔴：最终洞察"用AI要建立反馈回路"——资深AI用户已经直觉性地在做。读者反应是"说得对"不是"我靠原来可以这样"。
- 分享货币 🔴：读者转给朋友时讲不清"控制工程三原则是什么"，转发不构成自我表达。分享货币为零。

杀。选题本身不可行——源域（控制工程）和目标域（AI使用）之间的认知距离超过了进化猫读者的接受半径。

### 组合规则

大多数文章混合 2 种契约。常见组合：

| 主导 | 辅助 | 效果 |
|------|------|------|
| 类比传导 | 认知反转 | 新角度 + 强记忆锚点 |
| 认知反转 | 类比传导 | 破旧 + 建新 |
| 故事见证 | 框架清单 | 感染 + 干货 |
| 悬疑解谜 | 类比传导 | 好奇心驱动 + 跨域洞察 |
| 类比传导 | 问题方案 | 理解 + 行动 |

**禁止：** 3种以上混合 → 读者契约混乱，不知道签了哪张合同。

## 四、契约检验（写完后自检）

```
□ 如果我是读者，读完能用一句话说出"我得到了什么"吗？
□ 这篇文章从第一段就在回答合同上的问题吗？（不是从第三段才开始）
□ 文章最后一段兑现了第一段挂的问号吗？（不是烂尾）
□ 有没有哪一段跟合同无关？（有 → 删）
```
