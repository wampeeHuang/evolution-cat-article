# 相对时间一致性检查

> 血肉层，可通过 retro 回流更新。
> 职责：绝对日期 → 相对时间表达，逐条对照 `currentDate`。

---

## 为什么需要

事实校验验证绝对日期（"文章 2025-07-17 发表" ✅），但写作时翻译成"今年七月/去年七月/上个月"这类相对表达时，Agent 不会自动对照当前日期。2026-06-04 实战暴露："2025年7月"被写成"今年七月"——当前是2026年6月，2025年7月是"去年七月"。

根因不是搜索不充分，是**写作翻译环节缺少时间锚点检查**。

---

## 检查规则

| 相对表达 | 对照 `currentDate` 的判定条件 |
|---------|------------------------------|
| "今年 X 月" | 年份 = currentDate.year |
| "去年 X 月" | 年份 = currentDate.year - 1 |
| "上个月" | 月份 = currentDate.month - 1（跨年另行计算） |
| "本周/上周" | 所在周 = currentDate 所在周 / 前一周 |
| "最近" | 时间跨度 ≤ 90 天 |
| "即将" | 未来 30 天内 |
| "XX 年前" | currentDate.year - 事件年份 = XX |

---

## 执行时机

**Step L3 机械扫（轻输出）或 Step 3 机械扫（深度输出）**：逐句扫描文中所有相对时间表达，逐条对照 `currentDate`。

**特殊处理**：
- "这可能是今年最被低估的文章" → "今年"指 currentDate.year，主观判断，不触发
- "2026 年 IEEE 候任主席" → 绝对年份，不触发（已由事实校验覆盖）
- "在 Honeywell 干了 30 年" → 时长表达，来源已标注，不触发

**触发条件**：相对时间词（今年/去年/本月/上个月/本周/上周/最近/即将/N年前）+ 指向外部事件（非主观判断）。

---

## 自检

写完逐句过：文中每个"今年/去年/上个月/N年前"——拿 `currentDate` 算一遍，对得上才放行。
