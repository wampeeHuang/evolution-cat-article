判

阶段一：选题判断 + 内容分类

加载：flesh/topic-taxonomy.md + skeleton/kill-gate.md §一
判定内容基因（5 类型路由 --light / --deep）。运行第一道门禁（四问）：①认知税 ②洞察值 ③分享货币 ④平台合规。3+🔴→杀；④任一🔴→杀或重定角度。
→ 路由决策 + 门禁结果
🔴 第一道门禁 · 判
杀稿四问 3+ 红灯 → 杀稿；平台合规 🔴 → 杀或重定角度
① 认知税 ≥3 → 🔴 ② 洞察值 "嗯我知道" → 🔴 ③ 分享货币 讲不清楚 → 🔴 ④ 平台合规 🔴 → 杀
选题可行性 + 平台合规是初始条件。前三问全绿但平台违规，照杀。
搜

阶段二：深度搜索 + 意外发现

加载：skeleton/source-trace.md + fact-check.md + kill-gate.md §二
AI 透镜前置 → 6 维度并行搜索（每维追加 AI 变体） → 素材包落盘。读完整素材包提炼意外发现（≥3 条，每条引用素材行号）。运行第二道门禁：0 意外发现 → 🔴 杀。
→ article-source-pack.md + surprise-findings.md
🔴 第二道门禁 · 搜
0 条意外发现 → 必须杀稿
搜完没有意外 = 每一条都在验证已知。至少 1 个意外发现来自一手来源。
构

阶段三：结构构思 + 创意简报

加载：skeleton/reader-contract.md + flesh/writing-patterns.md
基于实际素材声明读者契约（8 类型选 ≤2 种混合）。输出 creative-brief（6 字段：意外发现 + 契约 + 递进链 + 舍弃清单 ≥3 项 + 翻牌点 + 一句话核心 ≤30 字）。
→ creative-brief.md（6 字段锁定）
写

阶段四：执行写作 + 事实核查

加载：skeleton/writing-craft.md + density-gate.md + flesh/voice-signals.md + opening/closing-patterns.md
严格按 creative-brief 蓝图写作：22 技法执行 + 开头/结尾模式选型 + 禁止词库避开 + 术语降维。写作时即遵守 formatting-convention.md。事实核查（搜索驱动，非推理）。输出 3 标题候选。
→ draft-v1.md + title-candidates.md
审

阶段五：四层质检 + 编辑判断

加载：humanizer-zh.md + flesh/avoid-list.md + skeleton/writing-craft.md §四
L1 硬性规则扫描 → L2 风格一致性 → L3 内容质量 → L4 活人感终审。标题裁定。运行第三道门禁：L4 FAIL → 🔴 杀；Signal/Noise 或上帝之声 FAIL → 🔴 杀（不给退回）。机械质检：反向映射 + 平台合规复核。
→ draft-v2.md + 质检报告
🔴 第三道门禁 · 审
L4 FAIL → 杀稿 · Signal/Noise FAIL → 杀 · 上帝之声 FAIL → 杀
"这篇东西读者在别处能拿到吗？"能 → 选题和角度层面，修不了。Signal/Noise 和上帝之声不是局部问题，是行文基调问题。
发

阶段六：配图 + 排版 + Forma 预览

加载：image-generation.md + formatting-convention.md
封面（900×383，不写标题文字）+ 配图 ≥2。终稿落盘。Forma 排版推送 → 拷贝图片 → 打开浏览器预览 → 验证就绪。复盘：向用户提 ≥3 个问题，信号写入 flesh/。
→ {标题}.md + 封面 + 配图 + Forma 预览
