1.5

意外发现暂停 · 杀稿门禁

主 session 自问
≥1 条意外发现 → 继续；0 条意外 → 杀稿。核心原则：线索→搜索→撞到意外→那才是真正的选题。没有意外 = 杀稿。
杀稿门禁
0 条意外发现 → 必须杀稿，不硬写
v4.0 最关键的变更。杀稿标准依赖 agent 判断力，可能假阳性/假阴性。
2

标题 + 诚实关

Beast + Karpathy · Gate 1+2
Beast 标题门禁(够不够炸) + Karpathy 诚关门禁(数字可追溯+壳核一致+不确定处标注)。双 PASS 才进 2.5
2.5

清晰度 + 大纲 + Surprising Idea

Feynman + PG · 三门禁
Feynman 清晰度守门(每个 section 能用日常语言讲清楚？) + PG 大纲门禁(Surprising Idea+递进链+论据追溯+逻辑断层)
3

开头 + 全文

Tim Pan + Sugarman
Tim Pan 开头(壳理论: 让意外发现变成钩子 + 技术术语配日常类比) + Sugarman 全文(滑滑梯节奏: 每句推着读者读下一句)
→ 正文草稿.md
门禁 · 开头硬指标
开头硬门禁 + 字数 + 引用可追溯
每 300 字 ≥1 个可验证数字或专有名词。术语首次出现：类比在前术语在后。
3.5

机械初审

Feynman · 仅错别字/句长/段长
不审内容质量，只审机械层面：错别字、句长超标、段长超标。机械通过→进 Step 4
4

去 AI 味

主 session · humanizer-zh.md
24 种 AI 模式检测 + grep 禁止词库。全文 0 命中 AI 高频词(此外/至关重要/标志着等)
硬阻断 · 双门禁
Step 5a 机械质检(5 项) + Step 5b 编辑判断(双签)
5a Ben Smith: 反向映射/高频词/字数/死链/落盘。5b BS+PG 双签: Signal/Noise + 承诺兑现 + 反面论证 + 上帝之声
5a

机械质检 · 5 项

Ben Smith
反向映射(每段是否可追溯到素材包) + AI高频词复检 + 字数合规 + 死链检查 + 文件落盘确认。5 项全过→进 5b
5b

编辑判断 · 双签

Ben Smith + Paul Graham
BS: Signal/Noise + 承诺兑现 + 反面论证 + 上帝之声。PG: 递进链还在吗？读者能说出三个收获吗？双 PASS = 发；BS ①②③ FAIL = 杀稿
杀稿门禁
Step 5b 双签 FAIL → 必须杀稿，不硬发
Signal/Noise 或 承诺兑现 或 反面论证 任一项 FAIL → 杀稿。v4.0 第二道杀稿门禁。
⏸ 人审暂停 · Step 5b 后
双签通过，配图前最后确认。配图 API 有成本。等用户明确确认
