# 排版工匠写作规范

> 写 markdown 时遵循此规范，进排版工匠（蓝底西文 swiss）后自动化处理。
> 此规范是**输出格式约束**——Step 1/L2 写作时即遵守，Step 5/L5 落盘前再核对。

## 文件结构

```
{文章文件夹}/
  original.md       ← 按本规范写的原始 markdown
  images/
    cover.png       ← 封面图，16:9 横版
    body-1.png      ← 正文配图1，16:9 横版
    body-2.png      ← 正文配图2（可选）
```

## Markdown 写作规则

### 封面图

```markdown
![](images/cover.png)

# 文章标题
```

第一行封面图，第二行空行，第三行 `# ` 标题。预处理自动把封面移到标题下方。

### 小标题（`--` 标记）

```markdown
--

发现了吗？这里有一个奇怪的反转
```

- `--` 独占一行
- 下一行标题文字，**不超过 40 字**。超过 → 被当作段落分隔，不生成标题
- 编号（01 02 03...）由 CSS 自动生成，**不手写**

### 正文换行

在 `。？！` 后自动换行，产生短句独立强调效果。

```markdown
有一件事正在悄悄发生。但大多数人还没注意到。

下一代工具的用户不是你。是 AI。
```

渲染后各短句独立成行。

**闭合符号不拆：** `）」》】` 不会和前一句分开。
```
正确：（不用感叹号、不以"你"开头。）    → 不断开
```

### 分隔线

全文**只用一条** `---`，放在进化猫签名和参考来源之间：

```markdown
这里是进化猫，陪你一起看世界✨

---

参考来源：
1. ...
```

正文段落之间**只用空行**，不用 `---` 或 `--`。

### 参考来源

```markdown
参考来源：
1. 作者 — 文章标题，来源，日期
2. 作者 — 文章标题，来源，日期
```

- 放在进化猫签名 + `---` 分隔线**之后**
- 序号化，纯文本无需 HTML 标签
- 样式（灰色 12px）由 Forma 进化猫主题 `markdownRules.references` 自动处理

### 图片

```markdown
![](images/body-1.png)
```

- 16:9 横版
- 氛围图，不画流程图/架构图
- 不放开头（封面已控氛围）
- 放在段落自然断点

## 写作惯例（手动判断）

### 关键词加粗

扫读读者 3 秒能抓到主线。**仅加粗核心概念和关键数字**——不是整句，不是整段。

```markdown
一类叫**推理**——怎么想。一类叫**知识**——是什么。

AI 比真人朋友更同意你的概率高出 **49%**。
```

规则：
- 核心概念首次出现 → 加粗（推理、知识、谄媚、互补性缺口、认知对齐、目标对齐）
- 关键数字 → 加粗（**49%**、**4B**、**58 分 vs 42 分**）
- 结论性短句 → 不加粗（留给排版引擎的自动断句）
- 小标题 → 不加粗（统一格式由 CSS 控制）


### 短句强调

充分利用自动断句，关键判断句独立成行：

```markdown
下一代工具的用户不是你。是 AI。

最好的工具，是你不需要操作的工具。
```

### 长段落呼吸

同一节内论述超过 5-6 段时，手动插一个空行制造停顿。不要两个空行——一个就够了。
