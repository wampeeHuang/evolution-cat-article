# 进化猫 · 文章写作与发布

> 从飞书选题到微信草稿箱，8 步全流程自动化。搜索 → 创意会三关 → 写文章 → 编辑初审 → 配图生成 → 终审 → 发布。

---

## 这是什么

Claude Code 里说一句「写文章」「发公众号」，Skill 自动接管全程。不是套模板填空，是从选题理解到微信草稿箱的全流程编排。

**8 步线性管道**，每步有质量门禁：

```
选题拉取 → 全网搜索 → 创意会三关 → 写文章 → 打磨(编辑+去AI味) → 生成配图 → 11项终审 → 微信草稿箱
```

一步不通过不进下一步。两层架构驱动：

| 层 | 内容 | 策略 |
|----|------|------|
| **铁律层** | 事实校验、信息密度、叙述诚实性、去AI味、阶段门禁、必落盘 | **不可跳过** |
| **节奏层** | 用什么结构、怎么开头、选什么节奏原型 | **每次由选题决定** |

## 快速开始

```bash
# 安装 skill 后，Claude Code 中直接说：
写文章
写公众号
把飞书入选选题写成文章
发草稿箱
```

首次运行会逐项询问：
1. 飞书 config.yaml 路径（非飞书模式可跳过）
2. 文章输出目录（默认 `./output`）
3. wechat-publisher skill 是否已安装

答案缓存到 `_runtime/paths.json`，后续不再问。

## 输入方式

- **飞书拉取**：自动从飞书表格拉取 `状态=已入选` 的选题
- **直接说出**：直接给选题 + 资料（链接、截图、文字），自动并入素材包

## 读者画像

| 维度 | 定义 |
|------|------|
| 身份 | 会使用 AI 产品的互联网人（产品/运营/设计/管理/学生），不是工程师 |
| 技术基线 | 知道 Token、API、开源是什么；不理解协议、框架、具体库名 |
| 阅读场景 | 微信公众号 + 手机竖屏，碎片时间 |
| 弃读原因 | 连续两个自然段看不懂 → 划走；满屏技术名词没解释 → 划走 |

## 创意会：三道串行门禁

写文章前，三道门禁串行审核——后者不改前者的决定。每道最多 2 轮：

| 顺序 | 门禁 | 负责人 | 做什么 | 裁决权 |
|------|------|--------|--------|--------|
| 第一道 | 标题关 | MrBeast | 极端对比/数字锚定/赌注型标题 | 标题否决 |
| 第二道 | 壳核关 | Tim 潘天鸿 | 标题↔封面↔内核互译检查 | 对齐否决 |
| 第三道 | 诚实关 | Karpathy | 有没有夸大/吹牛/编数据？ | **最高权重** |

三道全部通过 + 能一句话说清「读者看完会改变什么」→ 才进入写作。

## 三种节奏原型

| 原型 | 适用 | 结构 |
|------|------|------|
| 工具上手型 | 新产品/工具/方法 | 解决什么问题 → 别人实测效果 → 怎么复现 |
| 新闻解读型 | 热点事件/行业动态 | 发生了什么 → 为什么重要 → 对你意味着什么 |
| 观点论证型 | 反直觉判断/趋势预测 | 反直觉结论 → 证据 1/2/3 → 你的行动 |

全自动场景禁止故事叙述型（没有亲身经历可用），全文引用式叙述。

## 风格硬指标

| 指标 | 约束 |
|------|------|
| 字数 | 2000–6000 |
| 信息密度 | 每 300 字至少 1 个可验证数字或专有名词 |
| 最长句 | ≤ 45 字 |
| AI 高频词 | 0 命中 |
| 术语 | 类比在前术语在后，类比用日常场景 |
| 人设 | 趁午休给朋友安利好东西——不装专家 |

## 配图生成

三级降级策略：

| 优先级 | 方案 | 说明 |
|--------|------|------|
| 1 | gpt-image-2 API | 英文提示词，16:9 横版 |
| 2 | 即梦网页版 | API 失败后降级，浏览器自动化 |
| 3 | 提示词 txt | 前两级都失败，生成 txt 供人工处理 |

生图前查 [awesome-gpt-image-2](https://github.com/freestylefly/awesome-gpt-image-2) 案例库（397 cases），匹配视觉方向→提取提示词模式→填入文章内容。

## 终审 11 项

| # | 检查项 | # | 检查项 |
|---|--------|---|--------|
| 1 | 数据复核 | 7 | AI 味 0 命中 |
| 2 | 标题冲击力 | 8 | 配图 ≥ 2 张 |
| 3 | 封面图 | 9 | 参考来源可访问 |
| 4 | 字数 2000-6000 | 10 | 结尾进化猫签名 |
| 5 | 叙述诚实 | 11 | 文件落盘 |
| 6 | 术语降维 | | |

迭代上限 2 轮。2 轮后仍有不达标 → 标注交付，不无限打磨。

## 诚实边界

- **适用**：AI 工具上手、AI 新闻解读、AI 趋势观点
- **不适用**：纯技术教程、学术论文解读、访谈/人物特写
- **全自动限制**：禁止故事叙述型、禁止伪造第一人称体验
- **已知未覆盖**：多平台分发（小红书/知乎）、发布后数据追踪
- **信息截止**：Skill 制定于 2026-05，视角提炼自 2025 年公开资料
