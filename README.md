# 进化猫 · 文章写作与发布

把选题（飞书拉取或直接说出）变成微信草稿箱里的可预览文章——全流程自动化。

## 架构

一条 7 段线性管道，每段有质量门禁：

```
选题 → 全网搜索 → 创意会 → 写文章 → 打磨 → 生成配图 → 终审 → 微信草稿箱
```

**两层架构**：
- **铁律层**（不可跳过）：事实校验、信息密度、叙述诚实性、去 AI 味、阶段门禁、必落盘
- **节奏层**（按选题决定）：用什么结构、怎么开头、选什么节奏原型

## 质量系统

**4 个角色视角执行质量门禁**（构建者不自评）：

| 角色 | 负责 | 决策权重 |
|------|------|---------|
| MrBeast | 标题冲击力 | 标题否决权 |
| Tim 潘天鸿 | 壳核对齐 | 对齐否决权 |
| Karpathy | 事实诚实 | **最高权重**（争议时最终裁决） |
| Humanizer-zh | 去 AI 写作痕迹 | 风格否决权 |

**风格约束全部可验证**：字数 2000-6000、最长句 ≤ 45 字、AI 高频词命中 = 0、每 300 字至少 1 个可验证数字。

**迭代上限**：每道门禁最多 2 轮，超限标注交付，不无限打磨。

## 文件结构

```
SKILL.md                          # 执行入口（~280 行）
references/perspectives/          # 4 个写作视角（按需加载）
  ├── mrbeast-perspective.md      #   MrBeast 标题方法论
  ├── tim-pan-perspective.md      #   Tim 壳理论 + HKRR
  ├── karpathy-guidelines.md      #   Karpathy 诚实准则
  └── humanizer-zh.md             #   中文去 AI 味
.gitignore                        #   忽略 _runtime/
```

## 输入方式

- **飞书拉取**：从飞书表格自动拉取状态=「已入选」的选题
- **直接说出**：直接给选题 + 资料（链接、截图、文字均可），自动并入素材包
- **config.yaml**（飞书 API + image API 配置）；非飞书模式可跳过飞书配置
- wechat-publisher skill（发布到微信草稿箱）
- gpt-image-2 API 或即梦网页版（配图生成）

首次运行时逐项询问，结果缓存到 `_runtime/paths.json`。

## 诚实边界

- **适用**：AI 工具上手、AI 新闻解读、AI 趋势观点
- **不适用**：纯技术教程、学术论文解读、访谈/人物特写
- **全自动限制**：禁止故事叙述型节奏、禁止伪造第一人称体验
- **信息截止**：Skill 制定于 2026-05，写作视角提炼自 2025 年公开资料
- **已知未覆盖**：多平台分发（小红书/知乎）、发布后数据追踪
