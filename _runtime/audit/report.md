## 最弱 2 个维度

### 维度 10: 层层外化门禁 ★★☆ — 最弱

**问题**: 三道杀稿门禁是质量体系的基石，但只有第二道（意外发现）产生了文件化产物。第一道和第三道的 PASS/FAIL 判词只存在于对话上下文。

**具体表现**:
- 第一道门禁（选题四问）：判完即过，不留 `gate-1-result.md` 或等效文件
- 第三道门禁（L4 活人感）：审完即过，不留 `gate-3-result.md`
- 如果 Agent 跳过门禁直接写稿，无法事后检测

**改进建议**:

在 production-sop.md 阶段一末尾增加:

```markdown
>> 输出 _runtime/gate-1-result.md：
   - 四问逐项判定（🔴/🟡/🟢 + 一句话理由）
   - 总红灯数 + 最终判定（过/退回/杀）
   - 若 1🔴：标注补强方向
```

在 production-sop.md 阶段五末尾增加:

```markdown
>> 输出 _runtime/gate-3-result.md：
   - L4 四个感知维度逐项判定
   - Signal/Noise 扫描结果（0-signal 句数 + 位置）
   - 上帝之声扫描结果（命中数 + 位置）
   - 最终判定（PASS/FAIL/RETRY）
```

并在阶段六开头增加前置检查:

```markdown
>> 前置：_runtime/gate-1-result.md + gate-3-result.md 存在且均为 PASS
   任一缺失或 FAIL → 硬阻断
```

**为什么重要**: 三道门禁是这个技能最核心的质量防线。如果防线本身没有文件化痕迹，就等于没有防线。框架红线 9 明确禁止"无文件化的门禁"。

### 维度 3: 验证体系 ★★☆ — 第二弱

**问题**: 写作者、质检员、门禁裁判是同一个 Agent。没有独立于创作过程的验证机制。

**具体表现**:
- 同一模型生成文本 → 同一模型扫描 AI 模式 → 同一模型判断"读起来像活人吗"
- AI 写作的已知盲区（如过度自信、模式自噬）会同时影响创作和评判
- 复盘是唯一的外部验证，但：(1) 依赖用户主动参与 (2) 非结构化 (3) 滞后（发布后）

**改进建议**:

1. 在阶段五增加 spawn 子 agent 独立质检步骤:

```markdown
>> spawn 子 agent，加载 humanizer-zh.md + avoid-list.md + density-gate.md
>> 子 agent 独立扫描 draft-v1.md，输出 _runtime/independent-l1-scan.md
>> 主 agent 的 L1 扫描结果与子 agent 对比
   不一致项 → 标注 "主/子 agent 判定冲突"，人工裁定
```

2. 在 SKILL.md 诚实边界中增加:

```markdown
- **自评盲区**: 同一模型创作+质检存在模式自噬风险。AI 倾向给 AI 文本打高分。
  当前缓解措施: L1 硬性扫描（禁止词/格式）不受此影响；L4 主观判断可能偏松。
```

**为什么重要**: "AI 写 + AI 审"的闭环是内容生成技能的阿喀琉斯之踵。如果评判者与创作者共享同样的统计偏好，那么禁止词扫描以外的质检本质上不可靠。

## 领域追问

> Phase 1.6: 写作/内容管线领域的专家视角

1. **风格漂移无检测** [逻辑推演]: voice-signals 有 15 条信号 + 驱逐机制，但没有跨文章的风格一致性度量。第 10 篇文章和第 1 篇文章的"进化猫声音"可能已经不是同一个人。建议: 每 5 篇做一次风格回溯，对比最早和最新的 voice-signals 命中分布。

2. **门禁反复触发 → 上游策略问题** [逻辑推演 + 实战]: kill-gate.md 的杀稿示例显示"控制工程三原则"触发了第一道门禁 3🔴。如果同类选题反复触发门禁，说明 topic-taxonomy.md 的分类边界需要收紧——不是门禁的问题，是选题路由的问题。当前框架缺少"门禁触发频率监控"。

3. **血肉层初始信号来源单一** [逻辑推演]: voice-signals.md 的 15 条信号来自有限几篇文章（DeepSeek-Harness 占了 S001-S006，截图落盘占了 S010-S012，日记素材占了 S013-S015）。样本量小 + 时间集中 → 可能过度拟合特定类型文章的风格。建议: 诚实标注每个信号的泛化风险。
